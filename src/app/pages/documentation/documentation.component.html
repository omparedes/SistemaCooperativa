Arquitectura de Base de Datos (ACID)

Toda la lógica financiera crítica se ejecuta directamente en PostgreSQL (Supabase) mediante Stored Procedures (RPCs) con el patrón SECURITY DEFINER. Ante cualquier error de red o validación, la transacción completa hace ROLLBACK automático — no quedan estados intermedios.

Soft Deletes

Ningún registro financiero se elimina físicamente. Se usa deleted_at + anulado_por + motivo_anulacion.

Row Level Security

Cada tabla tiene políticas RLS por rol: Administrador, Caja y Asistente. El rol se valida en BD, no en el frontend.

Triggers Automáticos

Stock de inventario, saldos bancarios, auditoría de cambios y updated_at se gestionan automáticamente en BD.
